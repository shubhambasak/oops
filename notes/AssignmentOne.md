## Q1. Describe JVM and bytecode in Java architecture. Why is Java called platform independent?

### Java Virtual Machine (JVM):

JVM is a virtual machine that executes Java programs. It acts as an intermediary between Java code and the operating system. When a Java program is compiled, it is not converted directly into machine code. Instead, it is converted into bytecode, which is executed by the JVM.

### Main responsibilities of JVM:

- Loads Java class files into memory.
- Verifies bytecode for security.
- Converts bytecode into machine-specific instructions using JIT (Just-In-Time) compiler.
- Manages memory using Garbage Collection.
- Provides runtime environment for Java applications.

### Bytecode:

Bytecode is the intermediate code generated by the Java compiler (javac). It is stored in .class files and is not specific to any hardware or operating system. JVM interprets or compiles this bytecode into native machine code at runtime.

### Why Java is Platform Independent:

Java follows the principle "Write Once, Run Anywhere (WORA)".

The same bytecode can run on any system that has a JVM installed, whether it is Windows, Linux, or Mac. Since JVM is platform-specific but bytecode is not, Java programs become platform independent.

### Example:

A Java program compiled on Windows can run on Linux without recompilation if JVM is present.

## Q2. Define the concept of Classes and Objects in Java with a suitable example.

### Class:

A class is a blueprint or template used to create objects. It defines properties (variables/attributes) and behaviors (methods/functions) that the objects created from it will have.

A class does not occupy memory until an object is created.

```java
class ClassName {
    // variables (attributes)
    // methods (functions)
}
```

Object:

An object is an instance of a class. It represents a real-world entity and occupies memory when created.

Objects contain:

· State → represented by variables
· Behavior → represented by methods

Syntax of Object Creation:

```java
ClassName obj = new ClassName();
```

Here:

· new keyword allocates memory
· Constructor initializes the object

Real-Life Example

Let's create a class Car.

```java
class Car {
    String brand;
    int speed;

    void display() {
        System.out.println("Brand: " + brand);
        System.out.println("Speed: " + speed);
    }
}

public class Main {
    public static void main(String[] args) {
        Car c1 = new Car(); // Object creation
        c1.brand = "Toyota";
        c1.speed = 120;
        c1.display();
    }
}
```

Output:

```
Brand: Toyota
Speed: 120
```

Q3. Explain Interfaces in Java with a suitable example.

Interface in Java:

An interface in Java is a blueprint of a class that contains abstract methods (methods without body) and constants. It is used to achieve full abstraction and multiple inheritance in Java.

An interface specifies what a class must do, not how it does it.

Key Features of Interface:

1. All methods in an interface are public and abstract by default.
2. All variables are public, static, and final (constants).
3. A class uses the keyword implements to inherit an interface.
4. Java supports multiple inheritance using interfaces.
5. An interface cannot be instantiated (no objects directly).

Syntax of Interface:

```java
interface InterfaceName {
    void methodName();
}
```

Example: Interface Demonstration

```java
interface Animal {
    void sound(); // abstract method
}

class Dog implements Animal {
    public void sound() {
        System.out.println("Dog barks");
    }
}

class Cat implements Animal {
    public void sound() {
        System.out.println("Cat meows");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();
        Animal a2 = new Cat();

        a1.sound();
        a2.sound();
    }
}
```

Output:

```
Dog barks
Cat meows
```

Q4. Explain abstraction and abstract classes in Java. Describe abstract methods. Write a suitable example demonstrating abstraction using abstract classes.

Abstraction in Java:

Abstraction is the process of hiding implementation details and showing only the essential features of an object. It helps in reducing complexity and increases security by exposing only necessary information.

In Java, abstraction can be achieved using:

· Abstract classes
· Interfaces

Abstract Class:

An abstract class is a class that cannot be instantiated and may contain abstract methods (methods without body) as well as concrete methods (methods with body).

Key Points:

1. Declared using abstract keyword.
2. Cannot create objects directly.
3. Can contain both abstract and non-abstract methods.
4. Used as a base class for other classes.

Syntax:

```java
abstract class ClassName {
    abstract void methodName();
}
```

Abstract Method:

An abstract method is a method declared without implementation. It must be implemented by the subclass.

Syntax:

```java
abstract void display();
```

Example: Demonstrating Abstraction

```java
abstract class Shape {
    abstract void draw(); // abstract method

    void message() { // concrete method
        System.out.println("This is shape class");
    }
}

class Circle extends Shape {
    void draw() {
        System.out.println("Drawing Circle");
    }
}

class Rectangle extends Shape {
    void draw() {
        System.out.println("Drawing Rectangle");
    }
}

public class Main {
    public static void main(String[] args) {
        Shape s1 = new Circle();
        Shape s2 = new Rectangle();

        s1.draw();
        s2.draw();
        s1.message();
    }
}
```

Output:

```
Drawing Circle
Drawing Rectangle
This is shape class
```

Q5. Explain inheritance in Java with example. Why doesn't Java support multiple inheritance directly?

Inheritance in Java:

Inheritance is an Object-Oriented Programming concept in which one class acquires the properties and behaviors of another class. It promotes code reusability and establishes a relationship between classes.

The class whose properties are inherited is called the superclass (parent class) and the class that inherits is called the subclass (child class).

Java uses the keyword extends for inheritance.

Types of Inheritance in Java:

1. Single Inheritance
2. Multilevel Inheritance
3. Hierarchical Inheritance

(Java does not support multiple inheritance through classes.)

Syntax:

```java
class Parent {
    // properties and methods
}

class Child extends Parent {
    // inherited features + new features
}
```

Example of Inheritance:

```java
class Animal {
    void eat() {
        System.out.println("Animal eats food");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.eat();  // inherited method
        d.bark(); // own method
    }
}
```

Output:

```
Animal eats food
Dog barks
```

Explanation:

· Animal is the parent class.
· Dog is the child class.
· Dog inherits the method eat() from Animal.
· It also has its own method bark().

Why Java Doesn't Support Multiple Inheritance Directly:

Multiple inheritance means a class inheriting from more than one class.

Example (not allowed in Java):

```java
class A { }
class B { }
class C extends A, B { } // Error in Java
```

Reason: Diamond Problem

If both parent classes have the same method, the compiler gets confused about which method to use.

Example:

```java
class A {
    void show() { }
}

class B {
    void show() { }
}

class C extends A, B { } // ambiguity
```

To avoid such ambiguity and complexity, Java does not support multiple inheritance through classes.

However, Java supports multiple inheritance using interfaces.

Advantages of Inheritance:

1. Code reusability
2. Method overriding support
3. Improves program structure
4. Reduces redundancy

Q6. Illustrate polymorphism and its types in Java. Differentiate between runtime and compile-time polymorphism. Write a super class Shape with method displayArea() and subclasses Rectangle and Square. Demonstrate method overriding with a suitable example.

Polymorphism in Java:

Polymorphism means "many forms". It allows a single method or object to behave differently depending on the context. It is one of the important features of Object-Oriented Programming.

Example: A person can be a student, employee, or teacher depending on the situation.

Types of Polymorphism in Java:

1. Compile-time polymorphism (Static polymorphism) – Achieved using method overloading.
2. Runtime polymorphism (Dynamic polymorphism) – Achieved using method overriding.

Compile-Time Polymorphism:

Occurs when multiple methods have the same name but different parameters.

```java
class Add {
    int sum(int a, int b) {
        return a + b;
    }

    int sum(int a, int b, int c) {
        return a + b + c;
    }
}
```

Runtime Polymorphism:

Occurs when a subclass provides a specific implementation of a method already defined in the parent class.

Difference Between Compile-Time and Runtime Polymorphism:

Compile-Time Polymorphism Runtime Polymorphism
Achieved by method overloading Achieved by method overriding
Decision made at compile time Decision made at runtime
Faster execution Slightly slower
Same class methods Parent-child class relationship

Program Using Shape, Rectangle and Square (Method Overriding)

```java
class Shape {
    void displayArea() {
        System.out.println("Area calculation of shape");
    }
}

class Rectangle extends Shape {
    int length = 10;
    int breadth = 5;

    void displayArea() {
        int area = length * breadth;
        System.out.println("Area of Rectangle: " + area);
    }
}

class Square extends Shape {
    int side = 4;

    void displayArea() {
        int area = side * side;
        System.out.println("Area of Square: " + area);
    }
}

public class Main {
    public static void main(String[] args) {
        Shape s;

        s = new Rectangle();
        s.displayArea();

        s = new Square();
        s.displayArea();
    }
}
```

Output:

```
Area of Rectangle: 50
Area of Square: 16
```

Q7. Illustrate constructors and their application in Java. Describe the types of constructors used in Java. Design a class with attributes roll number, name, branch and email. Write all-argument constructor for class Student and create two objects with their construction.

Constructors in Java

A constructor is a special member function of a class that is used to initialize objects. It is automatically called when an object is created.

Characteristics of Constructor:

1. Constructor name must be same as class name.
2. It does not have any return type (not even void).
3. It is called automatically when object is created using new keyword.
4. It is mainly used to initialize instance variables.

Types of Constructors in Java

1. Default Constructor – Provided by compiler if no constructor is written. Does not take any arguments.
2. No-Argument Constructor – Written by programmer but takes no parameters.
3. Parameterized Constructor – Takes parameters. Used to initialize object with specific values.
4. Copy Constructor (User-defined) – Used to copy values from another object.

Application of Constructor

Constructors are used to:

· Initialize object data members.
· Allocate resources.
· Ensure object is created in a valid state.

Program: Student Class with All-Argument Constructor

```java
class Student {
    int rollNumber;
    String name;
    String branch;
    String email;

    // All-argument constructor
    Student(int rollNumber, String name, String branch, String email) {
        this.rollNumber = rollNumber;
        this.name = name;
        this.branch = branch;
        this.email = email;
    }

    void display() {
        System.out.println("Roll Number: " + rollNumber);
        System.out.println("Name: " + name);
        System.out.println("Branch: " + branch);
        System.out.println("Email: " + email);
        System.out.println();
    }
}

public class Main {
    public static void main(String[] args) {
        Student s1 = new Student(101, "Rahul Sharma", "CSE", "rahul@gmail.com");
        Student s2 = new Student(102, "Priya Singh", "IT", "priya@gmail.com");

        s1.display();
        s2.display();
    }
}
```

Output:

```
Roll Number: 101
Name: Rahul Sharma
Branch: CSE
Email: rahul@gmail.com

Roll Number: 102
Name: Priya Singh
Branch: IT
Email: priya@gmail.com
```
